Structure du projet — Cuisine du Monde

Document de présentation visuelle pour expliquer clairement toutes les pages du site, la logique MVC et l’organisation générale du projet.

1. Vue d’ensemble

Le projet est organisé en front-office pour les visiteurs et utilisateurs, et en back-office pour l’administrateur. L’accès aux pages dépend du rôle connecté.

PHP MySQL MVC HTML / CSS JavaScript

2. Rôle des dossiers

  • app/controllers : logique des pages
  • app/models : requêtes base de données
  • app/Views : affichage HTML/PHP
  • config : connexion base de données
  • public : point d’entrée du site
  • data : données CSV / fichiers sources

4. Toutes les pages du projet

Pages publiques

  • home.php : page d’accueil
  • login.php : connexion
  • register.php : inscription
  • forgot_password.php : mot de passe oublié
  • contact.php : page contact / contenu visuel
  • categories.php : choix des continents

Pages utilisateurs

  • recettes.php : liste des recettes
  • recipe_detail.php : détail d’une recette
  • category_recipes.php : recettes filtrées par catégorie

Pages administrateur

  • create_recipe.php : ajouter une recette
  • edit_recipe.php : modifier une recette
  • users.php : liste des utilisateurs
  • create_user.php : ajouter un utilisateur
  • edit_user.php : modifier un utilisateur

Parties communes

  • navbar.php : menu du site
  • footer.php : pied de page

5. Flux de fonctionnement MVC

1. Navigateur

  • L’utilisateur clique sur une page
  • Exemple : `index.php?page=recettes`

2. Controller

  • Le contrôleur reçoit la demande
  • Il décide quelle vue afficher

3. Model

  • Le modèle récupère ou modifie les données
  • Connexion à MySQL

4. View

  • La vue affiche la page HTML
  • Retour au navigateur
