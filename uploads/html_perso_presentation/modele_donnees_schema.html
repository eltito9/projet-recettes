Schéma du modèle de données

Projet ADB — Cuisine du Monde
Document visuel prêt à imprimer en PDF pour présenter l’analyse des données, le MCD, le MLD et le MPD.

Analyse rapide

Le projet repose sur deux entités principales :

  • users : comptes utilisateurs et administrateurs
  • recipes : recettes du site
PHP MySQL PDO MVC

Idée métier

Un utilisateur s’inscrit, se connecte et peut consulter les recettes. L’administrateur gère le contenu et les comptes. Le modèle de données doit donc permettre :

  • la gestion des identifiants et rôles
  • le stockage sécurisé des mots de passe
  • le stockage des informations de recettes

Remarque pour le jury : dans l’implémentation actuelle, la table recipes ne contient pas de clé étrangère user_id. Le schéma ci-dessus montre le besoin métier attendu ; tu peux dire que c’est une amélioration possible si on veut relier chaque recette à son auteur.

2. Modèle logique (MLD)

Tables présentes dans la base :

  • users(id_user, nom, prenom, username, password, role, created_at)
  • recipes(id, title, description, image, preparation_min, cuisson_min, calories_kcal, proteines_g, difficulte, categorie, created_at)

3. Modèle physique (MPD)

Structure SQL réellement utilisée dans le projet :

  • base de données projet_recettes
  • type de base : MySQL
  • connexion via PDO
  • mots de passe hashés avec password_hash()

4. Schéma relationnel simplifié

Table Clé primaire Champs principaux Rôle
users id_user nom, prenom, username, password, role, created_at Stocke les comptes visiteurs, utilisateurs et admin
recipes id title, description, image, preparation_min, cuisson_min, calories_kcal, proteines_g, difficulte, categorie, created_at Stocke les recettes affichées sur le site
Phrase simple à dire à l’oral : le modèle de données est construit autour de deux tables, une pour les utilisateurs et une pour les recettes, avec des champs adaptés à l’authentification et à l’affichage du contenu.

5. Conclusion à présenter

Cette analyse permet de comprendre comment les données sont organisées dans l’application. Le schéma montre les entités principales, leurs attributs, et la logique métier du site. C’est la base qui permet ensuite d’implémenter le CRUD, la connexion et la gestion des rôles.
