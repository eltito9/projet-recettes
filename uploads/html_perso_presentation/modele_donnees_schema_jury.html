Base : MySQL Architecture : MVC Sécurité : mots de passe hashés Tables principales : users / recipes

1) Lecture métier

Le site de recettes repose sur deux grands objets de données :

  • Utilisateur : compte de connexion, rôle et identifiants.
  • Recette : contenu affiché sur le site, avec image et informations nutritionnelles.

Le schéma ci-dessous sert à expliquer comment ces informations sont organisées dans la base.

Analyse Conception Base de données Modèle conceptuel

2) Ce que tu peux dire à l’oral

"J’ai analysé le besoin du projet puis j’ai identifié deux entités principales. Le modèle de données permet de gérer les comptes utilisateurs et les recettes. Chaque table contient des champs adaptés au fonctionnement du site et à la sécurité."

  • Un compte utilisateur peut être simple ou administrateur
  • Les mots de passe sont stockés en hash
  • Les recettes contiennent le titre, la description, l’image et les infos de préparation

Point important : dans le schéma métier idéal, un utilisateur peut créer plusieurs recettes. Dans l’implémentation actuelle, la base fonctionne déjà pour les comptes et les recettes, et la relation peut être ajoutée plus tard avec une clé étrangère user_id si on veut relier chaque recette à son auteur.

4) MLD

Le modèle logique reprend les tables réelles de la base :

  • users
  • recipes

5) MPD

Le modèle physique correspond à la structure SQL utilisée dans le projet :

  • MySQL / InnoDB
  • PDO côté PHP
  • champs typés et clés primaires

6) Sécurité

Le modèle prend en compte :

  • username unique
  • mot de passe haché
  • rôles user / admin

7) Structure relationnelle simplifiée

À mettre dans le dossier comme schéma clair
Table Clé primaire Champs principaux Rôle
users id_user nom, prenom, username, password, role, created_at Gérer les comptes et les droits d’accès
recipes id title, description, image, preparation_min, cuisson_min, calories_kcal, proteines_g, difficulte, categorie, created_at Stocker et afficher les recettes du site
Phrase courte pour le jury : le modèle de données est composé de deux tables principales, une pour les utilisateurs et une pour les recettes, ce qui permet de gérer la connexion, les rôles et le contenu du site de façon structurée.
